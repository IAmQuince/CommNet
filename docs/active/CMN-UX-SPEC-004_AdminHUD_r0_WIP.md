# Admin HUD Usability Specification

[SEC:CMN-UX-SPEC-004::PURPOSE]
The Admin HUD replaces the flat configuration grid with grouped operational zones. The target user should be able to determine LAN reachability, invite links, share state, user requests, mail status, and device verification from the first admin page.

[SEC:CMN-UX-SPEC-004::HUD-ZONES]
The HUD is organized into System Status, Quick Actions, Network & Invites, Sharing & Files, Users & Permissions, Mail, Devices & Catena, Portal Apps, and Diagnostics. Each zone uses a stable icon, domain color, status badge, and a small set of next actions.

[SEC:CMN-UX-SPEC-004::NAVIGATION]
The top navigation is intentionally reduced to major sections. Detailed tools are nested under section pages rather than being exposed as a long undifferentiated link strip.

[SEC:CMN-UX-SPEC-004::STATUS-LANGUAGE]
The UI avoids generic WORKING badges. State labels shall distinguish Ready, Needs setup, Warning, Simulated, Connected, Unverified, Guest visible, and Admin only.
