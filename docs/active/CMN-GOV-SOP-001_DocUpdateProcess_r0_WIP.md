# Controlled Document Update Process

| Revision | Date | Status | Summary |
|---|---|---|---|

## 1. Purpose [SEC:CMN-GOV-SOP-001::1]

This process prevents silent drift when CommNet documents, registries, requirements, and audit rules change.

## 2. Change Sequence [SEC:CMN-GOV-SOP-001::2]

1. Identify the reason for change.
2. Update the affected document revision history.
3. Update related registry records.
4. Update cross-reference records.
5. Run the package audit.
6. Review generated reports.
7. Record known debt if the package remains useful but incomplete.

## 3. No Silent Deletion Rule [SEC:CMN-GOV-SOP-001::3]

Documents, requirements, risks, and registry entries that influence design shall not be silently removed. They shall be superseded, archived, or closed with a reason.
