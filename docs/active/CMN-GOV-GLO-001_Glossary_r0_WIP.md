## 1. Purpose [SEC:CMN-GOV-GLO-001::1]

This glossary captures the terms that must remain stable across architecture, onboarding, security, privacy, and UI documents.

## 2. Glossary [SEC:CMN-GOV-GLO-001::2]

| Term | Definition |
|---|---|
| Admin console | Configuration surface for roles, permissions, privacy, services, devices, and diagnostics. |
| Audit event | Structured record of a security, policy, content, setup, or system action. |
| Community directory | Discoverable index of approved visible identities, nodes, services, or content. |
| Community web | User-facing local portal hosted by CommNet services. |
| Content item | A file, page, media item, tool, or resource registered with metadata. |
| Gateway boundary | Boundary crossed when a local node exposes or obtains service beyond the local trust area. |
| Local-first | Design principle that core functionality works offline and private by default. |
| Node role | Logical responsibility assigned to a device or process. |
| Policy profile | Exportable set of settings for privacy, permissions, roles, service exposure, and sharing. |
| Review state | Moderation/content status such as unreviewed, reviewed ok, restricted, blocked, broken, or admin review. |
| Runtime state | Local generated state not treated as package source. |
| Visibility | Discoverability of identity, node, service, or content. |
